# Voice Logger

PWA voice logger for nutrition and strength training. Voice-to-text via Web Speech API, parsed by Claude Haiku, nutrition lookup via Nutritionix API, logged directly to Google Sheets.

## Architecture

- **Frontend:** Vanilla JS PWA (dark theme, offline-capable shell)
- **Backend:** Python serverless functions on Vercel
- **Auth:** TOTP (6-digit code -> 30-min session token via HMAC)
- **Data:** Writes directly to Google Sheets (Nutrition tab, Strength Log tab)

## API Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | /api/health | No | Health check |
| POST | /api/health | No | TOTP login (exchange code for session token) |
| POST | /api/parse | Yes | Natural language -> structured data via Claude Haiku |
| POST | /api/lookup | Yes | Nutrition lookup via Nutritionix |
| POST | /api/log | Yes | Write nutrition or workout to Google Sheets |

## Environment Variables (Vercel)

| Variable | Purpose |
|----------|---------|
| TOTP_SECRET | Base32 secret generated by setup_totp.py |
| ANTHROPIC_API_KEY | Claude Haiku API key for natural language parsing |
| NUTRITIONIX_APP_ID | Nutritionix API application ID |
| NUTRITIONIX_APP_KEY | Nutritionix API key |
| SHEET_ID | Google Sheets workbook ID |
| GOOGLE_SERVICE_ACCOUNT_JSON | Base64-encoded service account JSON |

## CORS Configuration

CORS is configured in vercel.json for all API endpoints:

- **Origin:** * (all domains)
- **Methods:** GET, POST, OPTIONS
- **Custom headers:** Content-Type, Authorization, X-TOTP-Code
- Also handled in-code via `_shared.py` response headers

For production: restrict `Access-Control-Allow-Origin` to your specific domain.

## Setup

1. Run `python setup_totp.py` to generate a TOTP secret
2. Scan the QR code with Google Authenticator or Authy
3. Set all environment variables in the Vercel dashboard
4. Deploy with `vercel --prod`

## Rate Limiting

- 20 requests per session per hour (in-memory, resets on cold start)
- TOTP is primary auth; rate limiting is defense-in-depth
- For stricter limits, consider Vercel KV

## Project Structure

```
voice_logger/
  api/
    _shared.py        -- shared auth, CORS, rate limiting, Sheets client
    health.py          -- health check + TOTP login endpoint
    parse.py           -- Claude Haiku natural language parsing
    lookup.py          -- Nutritionix nutrition lookup
    log.py             -- write entries to Google Sheets
  static/              -- PWA frontend (HTML, JS, CSS, manifest)
  setup_totp.py        -- generate TOTP secret + QR code
  vercel.json          -- Vercel routing, CORS headers, static config
  requirements.txt     -- Python dependencies for serverless functions
```
